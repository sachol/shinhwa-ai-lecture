# 고객 응대·상담 보조 리서치 노트

> 작성: research-analyst | 검토: 미검증 | 최종 수정: 2026-05-28
> 영역: 공인중개사 고객 응대 AI 활용 (FAQ 템플릿, 카카오톡/이메일 답변 초안, 통화 요약, 다국어 응대)

---

## 0. 핵심 전제 — 개인정보 절대 원칙

공인중개사가 고객 응대에 AI를 쓸 때 가장 먼저 알아야 할 것은 **"고객의 식별 정보를 LLM에 그대로 넣지 말라"**이다. ChatGPT·Claude·Gemini 등 해외 LLM은 데이터가 미국 서버로 전송된다 → 개인정보 보호법 제28조의8(국외 이전) 동의 절차에 걸린다. **고객 동의 없이는 실명·연락처·주민번호·계약 상세를 그대로 입력할 수 없다.**

해결책은 두 가지:
1. **마스킹(가명처리)** 후 AI에 입력 (제28조의2 가명정보 활용)
2. **국내 LLM 사용** (네이버 HyperCLOVA X, 업스테이지 Solar 등 — 데이터가 국내 서버에 머무름)

---

## 1. 핵심 도구·서비스 (상위 7개)

### 1.1 ChatGPT (GPT-4o / GPT-5)
- **카테고리:** 범용 텍스트 생성 LLM
- **가격:** 무료 (제한적) / Plus 월 $20 / Pro 월 $200
- **한국어:** ★★★★★ (GPT-4o 이후 한국어 사실상 모국어 수준)
- **공인중개사 적합도:** ★★★★☆ (개인정보만 마스킹하면 최강)
- **요약:** 고객 카카오톡 답변 초안, 매물 추천 멘트, 영어·중국어 번역 응대 가능. 단 **데이터가 미국 서버로 송신**되므로 고객 실명·연락처는 마스킹 필수. 설정에서 "Improve the model for everyone" 토글을 끄면 학습 사용 차단 가능.
- **출처:** https://openai.com/policies/kr-privacy-policy/ (2026-05 확인); https://help.openai.com/ko-kr/articles/12677933-chatgpt-for-kakao-overview-and-kakao-tools (2026-05 확인)
- **입문자 핵심:** 설정 → 데이터 제어 → "모두를 위한 모델 개선" OFF부터 켜라. 고객 이름은 무조건 "고객A"로 바꿔라.
- **중급자 심화:** ChatGPT Team/Enterprise는 기본적으로 학습 OFF에 DPA(데이터처리위탁 계약) 체결 가능 → 사업자라면 법적 안전성 측면에서 Plus보다 Team이 유리.

### 1.2 Claude (Anthropic, Sonnet 4.5 / Opus 4.7)
- **카테고리:** 범용 텍스트 생성 LLM (장문 처리 강점)
- **가격:** 무료 / Pro 월 $20 / Team 1인당 월 $30
- **한국어:** ★★★★★
- **공인중개사 적합도:** ★★★★☆ (긴 카톡 대화 요약·통화록 요약에 강점)
- **요약:** 200K 컨텍스트로 카톡 일주일치 대화나 통화 녹취록 전체를 한 번에 분석 가능. ChatGPT보다 더 신중한 톤 → 법적 위험 표현(확정·보장) 자동 회피 경향.
- **출처:** https://www.anthropic.com/pricing (2026-05 확인)
- **입문자 핵심:** "이 고객 카톡 대화를 3줄로 요약하고 다음 답변 초안 작성" 식 활용. 마스킹 후 입력 원칙은 동일.
- **중급자 심화:** Claude Projects에 "FAQ 응대 가이드라인" 시스템 프롬프트를 고정해두면 매번 톤·규칙 재설명 불필요.

### 1.3 HyperCLOVA X (네이버 클라우드)
- **카테고리:** 한국어 특화 국산 LLM
- **가격:** 종량제 (네이버 클라우드 플랫폼)
- **한국어:** ★★★★★ (한국 데이터 6,500배 많이 학습)
- **공인중개사 적합도:** ★★★☆☆ (B2C 챗봇 클로바X는 2026.4.9 종료 → B2B용으로만)
- **요약:** **소비자용 클로바X는 종료**되었으나 기업용 HyperCLOVA X API는 유지. 데이터가 국내 서버에 머물러 국외이전 동의 부담이 없는 것이 최대 장점. 단 일반 공인중개사가 API를 직접 쓰기엔 진입장벽 있음 → 카카오톡 채널 연동 SaaS를 거쳐 사용하는 패턴 권장.
- **출처:** https://clova.ai/hyperclova (2026-05 확인); https://www.sentv.co.kr/article/view/sentv202603050087 (클로바X 종료, 2026-03 확인)
- **입문자 핵심:** "데이터 국내 보관" 메리트를 들었다면 클로바X가 아니라 **HyperCLOVA X 기업용**임을 알아야. 개인이 바로 쓸 수 있는 무료 챗봇은 더 이상 없다.
- **중급자 심화:** 네이버 클라우드 'CLOVA Studio'에서 프롬프트 엔지니어링·파인튜닝 가능 → 자기 부동산 사무소 톤매너 학습시킨 응대 봇 제작 가능.

## 2. 실제 적용 사례 (4개)

### 사례 2.1: 카카오톡 매물 문의에 답변 초안 자동 생성 (마스킹 패턴)
- **상황:** 강남구 역삼동 오피스텔 매물 문의가 카톡으로 들어왔다. 5건이 동시에 와서 일일이 답변하기 벅참.
- **사용 도구:** ChatGPT (Plus) + 수동 마스킹
- **마스킹 전 원문 (절대 그대로 입력 금지):**
  ```
  김민수 고객(010-1234-5678)이 역삼동 K오피스텔 1203호 매물 보고 싶다고 함.
  본인 회사 강남파이낸스센터 근무, 출근 도보 가능한지 물음.
  ```
- **마스킹 후 입력 (안전):**
  ```
  [고객A]가 강남구 역삼동 한 오피스텔 ###호 매물 문의함.
  근무지가 도보 가능 거리인지 궁금해함.
  공인중개사로서 정중하고 신뢰감 있는 카톡 답변 3개 톤(공손/친근/전문가)으로 작성.
  답변에는 "확정", "100%", "절대" 같은 단정 표현 빼고, 직접 방문 안내로 마무리.
  ```
- **출력 예시:** 3개 톤별 답변 + 방문 일정 제안 문구.
- **출처:** research-analyst 추정 시나리오 (실제 사례 미확보) — 마스킹 방법론은 개인정보보호위 안내서(2025.8) 원칙 적용

### 사례 2.2: 외국인 고객(영어·중국어) 응대 번역
- **상황:** 한남동 외국인 임차 문의가 영어로 들어옴. 답변은 가능하지만 계약서 조항 설명을 자연스럽게 영작하기 힘듦.
- **사용 도구:** ChatGPT 또는 Claude + 법무부 다국어 표준임대차계약서
- **활용법:**
  1. **법무부 공식 다국어 표준임대차계약서**(영어·중국어·베트남어) 다운로드 → 1차 출처
  2. 개별 질문에 대한 답변만 AI로 번역 (고객 이름 마스킹 필수)
  3. 한국 부동산 용어 정확성 검증: "전세=lump-sum lease deposit (Jeonse)", "관리비=maintenance fee" 등
- **프롬프트 예시:**
  ```
  Translate this Korean realtor message to natural, polite English for a foreign tenant.
  Keep Korean real estate terms (Jeonse, Wolse, Boong-Deok-Geum) in italics with English explanation.
  Original: "보증금 1억에 월세 80만원, 관리비 별도 15만원입니다. 다음 주 화요일 오후 2시 임장 가능하실까요?"
  ```
- **출처:** https://dazabi.com/insurance_magazine/article.php?id=9239 (법무부 다국어 표준계약서 배포, 2026 확인)
- **주의:** 계약서 본문은 반드시 법무부 공식 번역본 사용. AI 번역본을 그대로 계약 효력 문서로 쓰면 안 됨.

### 사례 2.3: 통화 종료 직후 자동 요약 → 다음 액션 정리
- **상황:** 임장 동행 후 30분 통화. 고객이 한꺼번에 6가지 질문(주차/엘리베이터/일조/소음/관리비/매도자 사정)을 함. 기록 누락이 두려움.
- **사용 도구:** 클로바노트 또는 에이닷 노트
- **워크플로우:**
  1. 통화 시작 시 "AI로 요약하기 위해 통화 녹음을 합니다" 안내 (1초)
  2. 통화 종료 → 클로바노트 앱에서 자동 텍스트화 + 주제별 요약
  3. 요약본을 Claude에 붙여넣어 "고객에게 보낼 정리 카톡 작성" (실명·전화번호 삭제 후)
  4. 카톡 발송 + 사무소 CRM에 요약 저장
- **출처:** https://devocean.sk.com/blog/techBoardDetail.do?ID=167794 (2026-05 확인)
- **법적 주의:** 일방 당사자(중개사 본인)가 참여한 대화 녹음은 통신비밀보호법 위반 아님. 다만 녹음 파일을 제3자(다른 사무소 등)와 공유하면 별도 위법 소지.

### 사례 2.4: 자주 묻는 질문(FAQ) 응답 템플릿 30개 생성
- **상황:** 신규 개업한 공인중개사. "취득세 얼마?", "전세보증보험 가입?", "특약 어떻게 쓰나?" 같은 단골 질문 답변 템플릿이 없음.
- **사용 도구:** ChatGPT + Claude 교차 검증
- **프롬프트:**
  ```
  당신은 서울 강남구 소재 10년차 공인중개사다.
  고객이 카톡으로 자주 묻는 질문 TOP 30개를 추출하고,
  각 질문에 대해 다음 형식으로 답변 템플릿 작성:
  - 질문: ...
  - 답변(3~5문장): 법적 책임 회피 표현 포함, 정확한 수치 대신 "조회 후 안내" 등 안전 문구
  - 후속 안내: 방문/추가 자료 안내
  카테고리: 매매/전세/월세/세금/계약/이사
  ```
- **출력 후 작업:** Claude에 다시 넣어 "각 답변이 공인중개사법 제25조 확인설명 의무와 광고법 위반 우려 표현 없는지 검토" 요청.
- **출처:** research-analyst 추정 시나리오; 공인중개사법 제25조 조문 — https://law.go.kr/법령/공인중개사법/제25조 (2026-05 확인)

다음을 ChatGPT/Claude 등 해외 LLM에 **그대로** 입력하면 개인정보보호법 위반 소지:

### 3.2 관련 법령 조문 (정확 인용)

> **개인정보 보호법 제17조 (개인정보의 제공)**
> 개인정보처리자는 정보주체의 동의를 받은 경우 등을 제외하고는 개인정보를 제3자에게 제공해서는 아니 된다.
> 출처: https://www.law.go.kr/법령/개인정보보호법/제17조 (2026-05 확인, 현행)

> **개인정보 보호법 제28조의2 (가명정보의 처리 등)**
> 개인정보처리자는 통계작성, 과학적 연구, 공익적 기록보존 등을 위하여 정보주체의 동의 없이 가명정보를 처리할 수 있다. 가명정보 처리 시 특정 개인을 알아보기 위한 목적으로 처리해서는 아니 된다.
> 출처: https://www.law.go.kr/법령/개인정보보호법 (2026-05 확인, 현행)
> **공인중개사 실무 적용:** 고객 정보를 ★마스킹(가명처리)★ 후 AI에 입력하면 가명정보로 분류 가능 → 동의 없이 활용 가능. 단 결합·역추적 시도 금지.

> **개인정보 보호법 제28조의8 (개인정보의 국외 이전)** — 2024.3.15 시행 개정
> 개인정보처리자는 다음 각 호의 어느 하나에 해당하는 경우 외에는 개인정보를 국외로 제공·처리위탁·보관할 수 없다:
> 1. 정보주체로부터 별도의 동의를 받은 경우
> 2. 법률·조약 등에 특별한 규정이 있는 경우
> 3. 계약 체결·이행을 위한 처리위탁으로서 개인정보 처리방침에 공개하거나 정보주체에게 알린 경우
> 4. 이전받는 자가 보호위원회 지정 인증을 받은 경우
> 5. 보호위원회가 보호 수준을 인정한 국가·국제기구로 이전하는 경우
> 출처: https://casenote.kr/법령/개인정보보호법/제28조의8 (2026-05 확인)
> **공인중개사 실무 적용:** ChatGPT(미국 OpenAI)에 고객 개인정보를 그대로 보내면 = 국외 이전. 고객 별도 동의 필요. **마스킹 후 가명정보로 변환하면 본 조항 적용 회피 가능.**

> **공인중개사법 제25조 (중개대상물의 확인·설명)**
> 개업공인중개사는 중개를 의뢰받은 경우 중개가 완성되기 전에 중개대상물의 권리관계, 법령 제한, 거래·이용 제한사항 등을 성실·정확하게 설명하고 토지대장 등본 또는 부동산 종합증명서, 등기사항증명서 등 설명의 근거자료를 제시하여야 한다.
> 출처: https://law.go.kr/법령/공인중개사법/제25조 (2026-05 확인)
> **AI 응대 함의:** AI가 생성한 답변을 그대로 고객에게 보내면 안 됨. **반드시 공인중개사가 검토 후 송부**해야 확인설명 의무 이행으로 인정된다. AI는 보조 도구일 뿐, 책임 주체는 사람.

### 3.3 마스킹 패턴 — 정규식(regex) 예시

공인중개사가 직접 코드 짤 일은 없지만, 카톡 대화에서 복사해온 텍스트를 AI에 넣기 전 다음 항목을 **수동 또는 도구로 치환**해야 한다. 참고용 패턴:

```python
import re

def mask_personal_info(text: str) -> str:
    # 1. 휴대전화 (010-1234-5678, 01012345678 등)
    text = re.sub(r'01[016789][-\s]?\d{3,4}[-\s]?\d{4}', '010-****-****', text)
    # 2. 주민등록번호 (앞 6자리만 살리고 뒤 마스킹)
    text = re.sub(r'(\d{6})[-\s]?\d{7}', r'\1-*******', text)
    # 3. 일반 전화 (02-1234-5678 등)
    text = re.sub(r'0\d{1,2}[-\s]?\d{3,4}[-\s]?\d{4}', '0##-****-****', text)
    # 4. 이메일
    text = re.sub(r'[\w\.-]+@[\w\.-]+\.\w+', '[email]', text)
    # 5. 계좌번호 (3-2-6 또는 3-4-7 등)
    text = re.sub(r'\d{3,6}[-\s]\d{2,4}[-\s]\d{4,7}', '[계좌]', text)
    # 6. 한국 이름 (2~4글자) — 자동화 어려움, 수동 치환 권장
    return text

# 사용 예
원문 = "김민수님(010-1234-5678) 보증금 2억 입금 우리은행 1002-345-678901"
print(mask_personal_info(원문))
# 출력: 김민수님(010-****-****) 보증금 2억 입금 우리은행 [계좌]
# → 이름은 별도로 "고객A"로 수동 치환
```

**입문자에게 권장:** 코드 안 짜도 됨. 카톡 원문을 메모장에 붙여넣고 Ctrl+H로 이름·전화번호 일괄 치환 → AI에 입력.

**중급자에게 권장:** 사무소 자체 CRM이 있다면 위 함수를 응대 자동화 파이프라인에 끼워넣어 LLM 호출 전 자동 마스킹.

### 3.4 환각(hallucination) 위험

AI가 그럴듯하게 만들어내는 거짓 정보:
- **취득세율** — 매년 바뀜. AI 답변 그대로 고객에게 보내지 말 것. 위택스·홈택스 원본 확인.
- **공시지가·실거래가** — AI는 최신 데이터 없음. 국토부 실거래가 공개시스템 사용.
- **법령 조문** — "제○조의○"를 AI가 만들어낼 수 있음. 법제처 국가법령정보센터 교차 확인.

### 3.5 챗봇 자동 응대의 한계

공인중개사법 제25조 확인설명 의무는 **인적 행위**다. 카카오톡 채널에 "AI가 자동 응대"하도록 설정해두면 다음 위험:
1. AI가 잘못된 정보 → 손해배상 책임은 개업공인중개사
2. 확인설명 의무 불이행으로 행정처분 대상
3. 광고법상 표시·광고 위반 가능

→ **자동 응대는 단순 안내(영업시간, 위치)까지만**. 매물·가격·계약 문의는 반드시 사람이 답변.
