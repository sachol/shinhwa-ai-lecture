ChatGPT 단독 vs 하네스

진짜 차이
차이점 ChatGPT 단독 하네스 (5 에이전트)
작업 깊이 얕음 표면적("일반론") 깊음 각 에이전트가 자기 영역만 파니까
사실 검증 수동 사람이 따로 검증 자동 qa-reviewer가 1차 출처 재검증
일관성 깨짐 모듈마다 다름 통일 표준 가상 매물·법령 인용 통일
재실행 처음부터 다시 부분 영향받은 에이전트만
비용 낮음 (월 $20) 중간 (Claude Code Pro + 토큰)
학습 곡선 즉시 사용 하네스 구성 1회 (이후 무한 재사용)
6

핵심 설계 결정 3가지

①

병렬 vs 순차

리서치 4개 영역은 의존 없음 → 병렬. 커리큘럼은 리서치 4건이 끝나야 시작 → 순차. 콘텐츠는 톤 일관성 위해 3명만 묶음 병렬.

②

파일 기반 데이터 전달

메모리·메시지가 아닌 _workspace/의 .md 파일로 주고받음. 사후 검증·부분 재실행 가능, 사람이 어디가 문제인지 진단 가능.

③

QA "경계면 교차 비교"

"슬라이드 다 있다"가 아니라 "리서치 노트의 도구가 콘텐츠 슬라이드에 정확히 등장하는가" 등 7개 경계면 검증. 1건만 어긋나도 Critical/Major.
